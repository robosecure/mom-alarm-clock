What This Demo Proves vs. What Needs Native Work

Demonstrated in This Web Prototype

  • Full alarm state machine (idle, active, verified, locked)
  • Escalation ladder with timed threshold crossings
  • Four distinct wake verification methods with validation logic
  • Morning Lock state with per-child config
  • Tamper attempt detection and logging across all cheat scenarios
  • Parent controls: nudge, force lock, grant access, preset toggle
  • Per-child configuration, streaks, on-time statistics, and history log
  • Bidirectional parent-child messaging
  • Fast-forward simulation for escalation testing
  • Persistent demo state via localStorage

Requires Native iOS / MDM Work

  • Screen Time API + FamilyControls framework to enforce app blocking
  • Device Management Profile (MDM) to resist uninstall without parent PIN
  • DeviceActivityMonitor to track app usage and enforce "allowed apps" list
  • Real-device motion sensors (CMMotionManager) for motion/steps verification
  • Camera access for actual photo-proof capture and upload
  • AVFoundation audio to enforce volume floor below system level
  • APNs push notifications for parent alerts at each escalation step
  • Background audio session to keep alarm ringing when app is closed
  • QR code generation + camera-based scanning (Vision framework)
  • iCloud / server sync for multi-device parent/child communication
